发表于 2018-05-13

区块链3.0架构是分布式架构，但可以不是完全区中心化的，最有可能是在不同场景下的混合结构，比如在特别需要消费者参与共识过程的环境下可能用区中心化的架构；而在很多企业使用场景，可能用半中心化（例如联盟链）或部分中心化（例如私有链）。

¶互联链架构剖析：

互联链是实现不同区块链互联互通的统一架构和标准协议。

互联账本不单包含了在以区块链为基础的账本之间的互联互通，也包含了多种账本（传统账本、区块链账本）之间的互联互通。

互联账本协议组：

每层只与邻层交互，不会跨层通信。

  • 账本层：为了支持账户间的转账，账本必须实现一些API和协议，称为账本层。这一层会有很多不同的协议，每种账本会对应一种协议。
  • 互联账本层：保证不同的连接器能互相操作，共同完成交易的路由。
  • 传输层：负责协调不同的转账交易，使它们能进行跨账本转账的交易。转账的安全性取决于采用的传输协议。
  • 应用层：负责谈判支付的以下关键属性：源账户、目标账户、金额、条件，当这些属性都定下来之后，就会调用传输层协议进行支付。
