区块链2.0架构：以太坊区块链

发表于 2018-05-07

区块链2.0架构把区块链作为一UTXO个可编程的分布式信用基础设施，支持智能合约应用。

  • 基本架构：

  • 账户设计：

    比特币没有账户的概念，每个用户的余额都是从他们在区块链上的UTXO计算出来的。以太坊的设计是将区块链作为一个通用的管理对象状态转换的去中心化平台，账户就是有状态的对象。外部账户的状态就是余额，而合约账户的状态可以是余额、代码执行情况，以及合约的存储。以太坊网络的状态就是所有账户的状态，该状态由每个区块的交易来更新，同时须在全网形成共识。

    外部账户可以通过发送交易到合约账户来实现对合同的调用。

  • 区块链设计：

    以太坊引入了账户的概念，带来更为复杂的校验和查询需求。以太坊区块头中存放了三个根哈希值：Merkle根、状态树根、收据树根。账户的状态由键值表来表示，在Patricia树中，键被编码成向下访问树的路径，值是账户里声明的变量（余额、nonce、代码、存储）。另一点不同于比特币的是，以太坊的区块链中的每个区块保存区块链号和区块难度。

  • PoW机制：

    以太坊的PoW算法为Ethash算法，概算法加入了内存难度，即算法中要求选择由随机数和区块头报文决定的一部分固定资源，这些资源叫“有向无环图（DAG）”，一般是几个GB。每3000个区块后需要有一个全新的DAG，称为一个epoch。因为DAG只和区块链深度有关，因此可以提前生成，客户端需要等生成了DAG后才能产生新区块，进行epoch转换时，若客户端没有预先生成DAG，网络会出现大规模的延迟。

    采用“GHOST”算法来构区块链，以太坊的区块链是一棵树，它包含“叔区块”。“叔区块”指符合难度条件，但区块里的交易不被确认的区块，即废块（stale）。产生“叔区块”的矿工和将“叔区块”包含进区块链的矿工都会得到奖励。

    “叔区块”是当前链接区块往前一推最多6个的“祖先”废块，每个区块最多能链接两个“叔区块”。

  • 计算和图灵完备：

    EVM是图灵完备的计算环境，即在EVM上可以做所有运算，包括无线循环，另外，一个合约可以调用其他合约，提供了潜在的递归调用。为了防止破坏者通过强制矿工或者全节点进入死循环而将他们基本关掉，以太坊采用了经济的方法保证安全—Gas。

  • EVM高级语言：

    以太坊提供高级语言供用户编写智能合约，程序会被编译成evm字节吗，部署在以太坊区块链上。

  • 以太坊P2P网络：

    • RLPx协议：专门为区中心化应用设计的，采用密码方式通信的P2P网络协议，为应用在P2P网络上的通信提供通用的传输和接口。

      RLP（递归长度前缀）是一种编码方式，其目的是将二进制数据进行任意嵌套的数组编码，在以太坊里，RLP用来对对象进行编码。

    • Whisper协议：DApp间的通信协议。是一个纯标志的系统标志，它提供一个低层次的（非应用相关的）但又简易使用的API，而不需要记忆底层的硬件属性。

      Whisper是为需要大规模的多对多数据发现 、信号谈判和最少的传输通信、完全的隐私保护的下一代DApp而设计的。

  • 事件：

    合约是在区块验证的时候被交易触发。合约与外部的通信是通过日志事件实现的。日志事件是交易执行时产生的收据的一部分。
